# AKS Newsletter – January 2026

January brings a strong mix of platform hardening, networking and identity improvements, and operational tooling for AKS. This edition focuses on what actually changes how clusters are built, operated, and secured. No hype. Just real platform progress.

Let's dive in.

---

## 🔎 Documentation Updates

* **[Configure kube-proxy](https://learn.microsoft.com/en-us/azure/aks/configure-kube-proxy)**: This refresh clarifies how kube-proxy is configured and operated in AKS, including supported modes and customization boundaries. It helps platform teams better understand when kube-proxy is still part of the datapath and how it interacts with newer networking stacks.

* **[External identity provider authentication – overview](https://learn.microsoft.com/en-us/azure/aks/external-identity-provider-authentication-overview)**: This documentation explains how AKS can integrate with external identity providers for control plane authentication. It is particularly relevant for organizations standardizing identity outside Microsoft Entra and operating multi-platform or partner-driven environments.

* **[Get started with AKS](https://learn.microsoft.com/en-us/azure/aks/get-started-aks)**: The onboarding flow was updated to better reflect current defaults and recommended cluster creation paths. It aligns the quick start with modern networking, identity, and observability guidance.

* **[AKS Kubernetes portal experience](https://learn.microsoft.com/en-us/azure/aks/kubernetes-portal?tabs=azure-cli)**: The portal documentation was updated to reflect the newer Kubernetes resource and workload management experiences. This is useful for teams relying on the portal for day-2 operations and troubleshooting.

* **[Local DNS customization](https://learn.microsoft.com/en-us/azure/aks/localdns-custom?tabs=enable)**: This update clarifies how LocalDNS can be configured and when it should be used instead of CoreDNS customization. It is important for clusters running large-scale or latency-sensitive workloads where DNS becomes part of the critical path.

* **[Managed Gateway API on AKS](https://learn.microsoft.com/en-us/azure/aks/managed-gateway-api?tabs=install)**: The documentation now better explains how to deploy and operate the managed Gateway API implementation on AKS. This supports teams planning a gradual migration away from traditional Ingress controllers.

* **[KMS migration from platform-managed to customer-managed keys](https://learn.microsoft.com/en-us/azure/aks/migrate-key-management-service-platform-managed-key-customer-managed-key)**: This guide documents how to migrate AKS key management from platform-managed to customer-managed keys without cluster recreation. It is a critical reference for regulated environments and security-driven organizations.

* **[Monitor AKS with Cilium dataplane](https://learn.microsoft.com/en-us/azure/aks/monitor-aks?tabs=cilium)**: The monitoring documentation now explicitly covers clusters using the Cilium dataplane. This helps close a common observability gap when moving away from the legacy networking stack.

* **[AKS support policies](https://learn.microsoft.com/en-us/azure/aks/support-policies)**: The support policy page was refreshed to reflect current versioning, upgrade expectations, and lifecycle boundaries. This is especially important for long-lived platform clusters.

* **[Configure Azure CNI networking](https://learn.microsoft.com/en-us/azure/aks/configure-azure-cni?tabs=configure-networking-portal)**: The Azure CNI configuration guide was updated to better reflect current networking models and supported scenarios. It improves clarity around advanced networking and integration with other AKS features.

* **[App Routing DNS and TLS](https://learn.microsoft.com/en-us/azure/aks/app-routing-dns-ssl)**: The documentation clarifies how DNS and certificate management work when using the built-in App Routing add-on. This helps simplify common ingress and exposure scenarios.

* **[Advanced AKS concepts](https://learn.microsoft.com/en-us/azure/aks/advanced-aks-concepts)**: This page was refreshed to consolidate advanced operational and architectural topics. It serves as a useful reference hub for platform and SRE teams.

## ✅ General Availability Announcements

* **[Deployment safeguards – Pod Security Standards support](https://learn.microsoft.com/azure/aks/deployment-safeguards#pod-security-standards-in-deployment-safeguards)**: AKS deployment safeguards now support Kubernetes Pod Security Standards. This allows enforcing baseline and restricted policies directly at deployment time, helping prevent insecure workloads from being introduced into the cluster.

* **[Ubuntu 24.04 support in AKS](https://aka.ms/aks/upgrade-os-version)**: Ubuntu 24.04 is now supported as a node OS option in AKS. This enables customers to align clusters with the latest Ubuntu LTS and benefit from a newer kernel and updated user space.

## 🧪 Preview Feature Announcements

* **[Azure CNI Overlay Pod CIDR expansion](https://learn.microsoft.com/azure/aks/azure-cni-overlay)**: Azure CNI Overlay now supports Pod CIDR address space expansion. This allows adding more Pod IPs without recreating the cluster, removing a long-standing operational limitation for large clusters.

* **[OpenTelemetry support for AKS monitoring (limited preview)](https://aka.ms/AzureMonitorOTelPreview)**: AKS now offers OpenTelemetry support for monitoring in a limited public preview. This enables standardized telemetry pipelines and simplifies integration with modern observability platforms.

* **[Private IP support for Static Egress Gateway](https://learn.microsoft.com/azure/aks/configure-static-egress-gateway#static-private-ip-support-preview)**: Static Egress Gateway now supports private IPs in preview for clusters running Kubernetes version 1.34 or newer. This enables tighter network isolation for outbound traffic scenarios.

* **[External identity provider authentication for the control plane](https://learn.microsoft.com/azure/aks/external-identity-provider-authentication-configure?pivots=github)**: AKS now supports external identity providers for cluster control plane authentication in preview. This is a major step for organizations that want to federate access using non-Entra identity systems.

* **[Identity bindings (preview)](https://learn.microsoft.com/azure/aks/identity-bindings-concepts)**: Identity bindings introduce a cleaner model for workload identity at scale. This addresses the 20 federated identity credential limit per managed identity and simplifies large-scale, multi-cluster identity designs.

* **[Flatcar Container Linux for AKS (preview)](https://aka.ms/aks/flatcar)**: Flatcar Container Linux is now available as a node OS option in preview. It is a vendor-neutral, immutable, container-optimized OS well suited for hybrid, multi-cloud, and on-prem aligned environments.

* **[Windows Server 2025 support (preview)](https://aka.ms/aks/upgrade-windows-os-version)**: Windows Server 2025 is now supported in preview. This release brings containerd 2.0 as default, generation 2 VMs by default, and FIPS enabled by default.

* **[Azure Linux with OS Guard (preview)](https://learn.microsoft.com/azure/azure-linux/intro-azure-linux-os-guard)**: Azure Linux with OS Guard introduces a hardened and immutable OS variant for AKS. It is designed for security-sensitive environments requiring stronger host protection.

* **[Istio CNI (preview)](https://learn.microsoft.com/azure/aks/istio-cni)**: Istio CNI is now available in public preview. It improves security by removing the need for NET_ADMIN and NET_RAW capabilities in application workloads running inside the mesh.

## 🔁 Behavioral Changes

* **podCIDR fields returned for networkPlugin=none**: Starting with API version 2026-01-01, AKS returns podCIDR and podCIDRs when networkPlugin is set to none. This allows customers to align pod address ranges with their chosen CNI configuration.

* **LocalDNS external forwarding restrictions**: When using [LocalDNS](https://learn.microsoft.com/azure/aks/localdns-custom), AKS now blocks forwarding external domains to CoreDNS from vnetDNSOverrides to prevent DNS resolution issues.

* **[Stricter subnet validation for networking add-ons](https://learn.microsoft.com/azure/application-gateway/for-containers/quickstart-create-application-gateway-for-containers-managed-by-alb-controller)**: AKS now enforces subnet requirements for networking add-ons such as Application Gateway for Containers. Misconfigured subnets can cause cluster creation or upgrade failures.

* **[Virtual network encryption with API server VNet integration](https://learn.microsoft.com/azure/aks/api-server-vnet-integration#limitations)**: AKS now returns a client error when virtual network encryption is used together with API server VNet integration. This configuration is not supported.
